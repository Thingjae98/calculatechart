# CLAUDE.md - CalculateChart 개발 컨텍스트

## 프로젝트 개요

한국 주식(KRX) 차트 기술적 분석 + 점수화 + N일 예측 웹 앱.
개발자(명재)가 **컴퓨터를 잘 모르는 가족(부모님, 누나)**에게 제공하기 위한 개인 프로젝트.

---

## ⚠️ 핵심 설계 원칙 — 비전문가 사용자 우선

> 이 앱의 사용자는 주식 차트를 처음 보는 엄마, 누나다.
> 기능이 많아도 화면이 복잡하면 쓸 수 없다.
> **단순함이 기능보다 항상 우선이다.**

### 반드시 지켜야 할 UI 원칙

1. **차트에 선은 최소화**: 동시에 보이는 수평선(지지/저항/이평선 등)은 **5개 이하**로 유지
2. **전문 용어 금지**: "RSI", "MACD", "다이버전스" 같은 용어는 UI에 절대 노출하지 말 것. 점수, 신호 카드에서는 쉬운 말로만 설명
3. **색상 의미를 직관적으로**: 초록=좋다, 빨강=위험, 노랑=관망. 이 외의 색상 의미 혼용 금지
4. **클릭 한 번으로 결론**: "지금 사도 되나요?"에 대한 답이 첫 화면에 바로 보여야 함
5. **기능 추가 전 항상 질문**: "엄마가 이걸 보고 뭘 해야 하는지 알 수 있나?" — YES이면 추가, NO이면 숨기거나 제거

### 복잡한 지표는 백엔드에서만 처리
스토캐스틱, 다이버전스, 일목균형표, 피보나치 등은 **점수 계산에만 활용**.
차트에 직접 표시하면 오히려 혼란을 주므로 시각화는 신중하게 결정.

## 아키텍처

- **프론트엔드**: React 19 + TypeScript + Vite. `lightweight-charts` v5로 캔들/거래량/이평선 렌더링.
- **백엔드**: FastAPI (Python). `pykrx`로 KRX OHLCV 데이터 취득, `scipy`로 지지/저항선 탐지.
- **배포**: Render.com 무료 플랜 (cold start 이슈 → `/api/ping` 웜업으로 대응). 프론트엔드는 Vercel.

## 핵심 파일

| 파일 | 역할 |
|------|------|
| `backend/main.py` | 전체 백엔드 (엔드포인트, 기술적 분석, 예측 캔들, 추천) |

## 개발 명령어

```bash
# 백엔드 (반드시 먼저 실행)
cd backend && uvicorn main:app --reload --port 8000

# 프론트엔드 (Vite 프록시가 /api/* → localhost:8000 전달)
cd frontend && npm run dev

# 빌드
cd frontend && npm run build
```

## 배포 설정

- **백엔드**: `render.yaml` → Render.com 자동 배포
- **프론트엔드**: Vercel에 `VITE_API_BASE_URL` 환경변수 필수 설정
- 로컬 개발 시 `VITE_API_BASE_URL`은 비워두면 Vite 프록시 사용

## 주의 사항

- `pykrx`는 동기 라이브러리 → `ThreadPoolExecutor`로 감싸서 비동기 호출 + 타임아웃 적용
- `pandas_ta`는 설치 실패 가능 → import 실패 시 수동 RSI 계산으로 폴백 (Wilder's EMA 방식 사용)
- 종목 리스트(`_load_listing`)는 fdr 우선 + pykrx 폴백, 1시간 캐시 + startup 사전 로드
- 추천 종목은 1시간 메모리 캐시. 캐시 없으면 220종목 순회하므로 수 분 소요
- CandleChart에서 `freshLoadId` prop으로 신규 로드 vs 스크롤 추가 로드를 구분
- 예측 캔들은 프론트에서 별도 CandlestickSeries(반투명)로 렌더링
- OHLCV 데이터에서 NaN/0값 행은 백엔드+프론트엔드 양쪽에서 필터링

## 🎨 차트 복잡도 문제 분석 및 해결 방향 (✅ 해결됨, commit 9e6d51e)

### 과거 문제: 최대 17개 수평선이 동시에 표시됨 → **현재는 기본 4선 이하로 축소**

| 레이어 | 개수 | 현재 상태 |
|--------|------|---------|
| 이동평균선 (5/20/60/120일) | 4선 | 항상 표시 |
| 지지선 / 저항선 | 2선 | 항상 표시 |
| 박스 상단/하단 | 2선 | 박스권 진입 시 표시 |
| 피보나치 레벨 | 5선 | 항상 표시 ← **즉시 제거 필요** |
| 일목균형표 (전환/기준/구름上下) | 4선 | 항상 표시 ← **즉시 제거 필요** |

비전문가가 보기에 선이 17개면 **무엇을 봐야 할지 알 수 없다**.

### 해결 방향: 기본 뷰 단순화 + 토글 방식

#### 기본 뷰 (항상 표시, 최대 5선)
| 표시 항목 | 이유 |
|---------|------|
| 캔들 + 거래량 | 필수 |
| **20일 이평선** (주황) | 한국 주식 방송에서 "20일선"으로 가장 많이 언급 |
| **60일 이평선** (파랑) | "2개월 추세선"으로 직관적 |
| 지지선 1개 (파랑 점선) | "여기서 버팀" — 의미 직관적 |
| 저항선 1개 (빨강 점선) | "여기서 막힘" — 의미 직관적 |

#### 숨김 처리 (토글로 ON/OFF)
| 숨길 항목 | 대안 |
|---------|------|
| 5일 / 120일 이평선 | "상세 이평선" 토글 버튼 |
| 피보나치 5선 | "전문가 모드" 토글 또는 완전 제거 |
| 일목균형표 4선 | 백엔드 점수 계산에만 활용, 차트 표시 제거 |
| 박스 상단/하단 | 박스권일 때만 표시 (현재 유지) |

#### 구현 방법 (CandleChart.tsx)
```tsx
// 상태 추가
const [showAllMA, setShowAllMA] = useState(false)
const [showAdvanced, setShowAdvanced] = useState(false)

// MA 렌더링 조건
// showAllMA=false: 20일, 60일만 표시
// showAllMA=true: 5일, 20일, 60일, 120일 모두

// 피보나치: showAdvanced=true일 때만 표시
// 일목균형표: 차트에서 완전 제거 (점수에만 활용)
```

#### 차트 위 토글 버튼 예시 (App.css와 통일)
```
[20일/60일 ✓]  [전체 이평선]  [피보나치]
```
버튼 스타일은 기존 `predBtn` 클래스와 동일하게 통일.

### 즉시 적용 우선순위
1. **피보나치 5선 → 기본 숨김** (토글로 선택)
2. **일목균형표 4선 → 차트에서 제거** (점수 계산만 유지)
3. **5일/120일 이평선 → 기본 숨김** (토글로 선택)
4. 나머지 (지지/저항/박스/20일/60일)는 유지

## 예측 모델 핵심 로직 (v3 — 몬테카를로 앙상블)

`_generate_predicted_candles()` 함수 (backend/main.py):

- **몬테카를로 앙상블**: 50개 경로 생성 → 중앙값으로 대표 캔들 도출
- **ADX + Hurst 지수**: 레짐(추세장/횡보장) 이중 판별
- **멀티타임프레임 모멘텀**: 5d/10d/20d/60d 가중 혼합
- **피보나치 자석 효과**: 0.382/0.618 레벨 내부 인력 적용 (차트 미표시)
- **적응형 GARCH**: 레짐별 변동성 파라미터 조정
- **앙상블 집계**: 고가=60th 퍼센타일, 저가=40th, 시/종가=중앙값
- **seed 결정론적**: `(last_close * 100 + date_hash)` 기반 — 같은 종목/날짜면 동일한 경로

## 지지/저항선 탐지 (v2)

- 다중 타임프레임(5/10/20봉) 피봇 고·저점 수집
- 터치횟수(40%) + 거래량집중도(30%) + 최근성(20%) + 현재가근접도(10%) 점수화
- 유효 범위: 현재가 위/아래 1%~20% 이내
- 가장 유의미한 1개씩만 최종 선택

## 수렴→돌파→눌림목 패턴 감지

- **돌파**: 최근 10일 내 BB 상단/박스 상단 돌파 + 거래량 20일 평균 1.8배 이상
- **눌림목**: 돌파 레벨 -3%~+5% 이내 되돌림 + 거래량 감소 → structure_pts +7
- **예측 반영**: 눌림목 구간에서 저항 반발력 1/3로 완화

## 커스텀 지표 설명

- **수박지표**: BB 스퀴즈 + 이평선 수렴(5/20/60/224일 편차 < 2%) 동시 발생 → 추세 전환 임박 신호
- **계단지표**: ATR×0.5 그리드 계단화. 3회+ 유지 + 상향 시 상승 확인

## 캔들 패턴 인식 (스코어링 반영, 최근 3봉)

| 패턴 | 점수 | 조건 |
|------|------|------|
| 망치형 🔨 | +3 | 전일 음봉 + 아래꼬리>몸통×2 |
| 역망치형 🔼 | +2 | 전일 음봉 + 위꼬리>몸통×2 |
| 도지 ⚖️ | 0~+2 | 몸통<ATR×0.1 |
| 상승장악형 🟢 | +3 | 전일 음봉을 양봉이 완전히 덮음 |
| 하락장악형 🔴 | -3 | 전일 양봉을 음봉이 완전히 덮음 |
| 모닝스타 🌟 | +5 | 큰음봉 + 소형봉 + 큰양봉 (3봉) |
| 이브닝스타 🌙 | -5 | 큰양봉 + 소형봉 + 큰음봉 (3봉) |
| 잠자리형 도지 🐉 | +3 | 긴 아래꼬리 + 도지 |
| 비석형 도지 🪦 | -3 | 긴 위꼬리 + 도지 |

## 점수 시스템 구조 (0-100, 기준 50)

| 카테고리 | 캡 | 주요 구성요소 |
|--------|-----|------------|
| 추세 | ±15 | MA 정렬, Cup 패턴, 골든크로스, **일목균형표** |
| 모멘텀 | ±12 | RSI, MACD, 이격도, **스토캐스틱**, 다이버전스 |
| 변동성 | ±10 | BB 위치, 수박지표(스퀴즈) |
| 거래량 | ±15 | 바닥 매집, OBV 다이버전스, 스퀴즈+거래량 폭발 |
| 구조 | ±8 | 지지/저항, 박스권, 계단지표, 눌림목, 캔들 패턴 |
| 반등 시너지 | +3~5 | 반등 플래그 2개+ 동시 발생 |

**이론상 최대합산 ~132점 → 100으로 클램핑됨**
**모든 지표는 점수에만 반영 — UI 신호 카드 문구는 반드시 쉬운 말로**

## 스토캐스틱 오실레이터 (14-3-3, 점수 전용)

- `%K` = 14일 범위 내 현재 위치, 3일 평활화 / `%D` = %K의 3일 평균
- RSI + 스토캐스틱 이중 과매도 → +5점 (강한 반등), 이중 과매수 → -5점
- %K가 %D 하향 돌파 + 바닥권: +3점 / 상향 돌파 + 고점권: -2점
- **차트에 표시하지 않음** — 점수 및 신호 문구에만 반영

## 일목균형표 (점수 전용)

- 전환선(9일), 기준선(26일), 선행스팬1·2, 구름(Kumo) 계산
- 구름 위 + 전환>기준 + 양운: +4점 / 구름 아래: -2~-4점
- **차트에 전환선/기준선/구름 선 현재 표시 중 → 기본 숨김으로 전환 예정**
- 스코어링에는 계속 활용

## 피보나치 되돌림 (점수 전용)

- 최근 120일 swing 고점/저점 기준 5레벨 (0.236~0.786) 계산
- 예측 모델 내 자석 효과에 활용, 스코어링에도 간접 반영
- **차트에 5선 현재 표시 중 → 기본 숨김으로 전환 예정**

## 다이버전스 감지 (점수 전용, find_peaks 기반)

`scipy.signal.find_peaks(distance=8, prominence=0.3σ)` 로 실제 피크 쌍 탐색:
- RSI 상승 다이버전스: +5점 / RSI 하락: -4점
- MACD 상승 다이버전스: +4점 / MACD 하락: -4점
- OBV 다이버전스: +8점
- **화면 노출 없음** — 신호 카드에 쉬운 문구로만 표시

## 추천 종목 필터링

- 시가총액 ≥ 500억 + 거래대금 ≥ 20억 (유동성 확보)
- 1시간 캐시, 220종목 샘플링

## 설계상 결정 (의도적)

- **예측 seed 결정론적**: 같은 종목/날짜 → 같은 예측 (재현성 확보)
- **SMA20 평균회귀**: `mean_revert += -dist_sma20 * 0.10` (올바른 방향)
- **스크롤 추가 로드**: OHLCV 데이터만 반환, fibonacci/ichimoku 미계산

### 🎯 다음 단계 우선순위 (B → C → A → D 권장)

**B. 시장 지수(KOSPI) 영향 차감 — 알파/베타 분해** ✅ **완료 (부분 개선 확인)**
- 구현 내용:
  - `_fetch_index_sync("KS11")` 추가 (fdr 기반 KOSPI 지수 조회)
  - `_generate_predicted_candles(market_df=...)` 옵셔널 파라미터 추가
  - 60일 log-return 기반 β 추정 (clamp [0.3, 2.0])
  - `alpha_mom = weighted_mom - β × market_weighted_mom`
  - 이후 drift 계산은 알파 모멘텀만 사용 → 시장 상승장 베타에 업혀 과대 예측되는 문제 완화
  - `/api/predict` 엔드포인트에서 KOSPI 병렬 조회 (`asyncio.gather`)
  - `backtest.py` 전역 KOSPI 1회 로드 후 walk-forward 각 anchor에 슬라이스 전달

- 백테스트 결과 (10종목 × 180일 = 1,080건):
  | 지표 | Baseline | Alpha/Beta | 변화 |
  |------|----------|-----------|------|
  | 1일 MAPE | 3.08% | 3.09% | ≈0 |
  | 7일 MAPE | 7.92% | **7.78%** | ↓ 1.8% 개선 |
  | 7일 DirAcc | 49.2% | **49.4%** | +0.2pp |
  | 7일 Bias | +1.52% | **+1.40%** | ↓ 상향편향 완화 |

  | 35-50 수익률 | +3.64% | +3.43% | 비슷 |

- **결론**:
  - MAPE/DirAcc/Bias 모두 약간 개선 (방향은 옳음 — 시장 베타 제거가 과대예측을 완화)
  - 그러나 **점수 역상관 문제는 여전** (75+ → 음수 수익)
  - 이는 시장 베타 문제가 아니라 **점수 시스템 자체의 철학 문제**임을 확정

  - 다음 단계는 점수 **가중치 리밸런싱**(모멘텀 가중 하향, 바닥 신호 가중 상향) 또는 **점수 해석 반전**
