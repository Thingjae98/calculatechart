# CLAUDE.md - CalculateChart 개발 컨텍스트

## 프로젝트 개요

한국 주식(KRX) 차트 기술적 분석 + 점수화 + N일 예측 웹 앱.
개발자(명재)가 가족(부모님, 누나)에게 제공하려는 목적으로 만든 개인 프로젝트.

## 아키텍처

- **프론트엔드**: React 19 + TypeScript + Vite. `lightweight-charts` v5로 캔들/거래량/이평선 렌더링.
- **백엔드**: FastAPI (Python). `pykrx`로 KRX OHLCV 데이터 취득, `scipy`로 지지/저항선 탐지.
- **배포**: Render.com 무료 플랜 (cold start 이슈 → `/api/ping` 웜업으로 대응). 프론트엔드는 Vercel.

## 핵심 파일

## 주의 사항

- `pykrx`는 동기 라이브러리 → `ThreadPoolExecutor`로 감싸서 비동기 호출 + 타임아웃 적용
- `pandas_ta`는 설치 실패 가능 → import 실패 시 수동 RSI 계산으로 폴백 (Wilder's EMA 방식 사용)
- 종목 리스트(`_load_listing`)는 fdr 우선 + pykrx 폴백, 1시간 캐시 + startup 사전 로드
- 추천 종목은 1시간 메모리 캐시. 캐시 없으면 220종목 순회하므로 수 분 소요
- CandleChart에서 `freshLoadId` prop으로 신규 로드 vs 스크롤 추가 로드를 구분
- 예측 캔들은 프론트에서 별도 CandlestickSeries(반투명)로 렌더링
- OHLCV 데이터에서 NaN/0값 행은 백엔드+프론트엔드 양쪽에서 필터링

## 예측 모델 핵심 로직 (v3 — 몬테카를로 앙상블)

`_generate_predicted_candles()` 함수 (backend/main.py):

### v2 → v3 핵심 변경점
- **몬테카를로 앙상블**: 50개 경로 생성 → 중앙값(median)으로 대표 캔들 도출 (단일 경로 운 의존 제거)
- **ADX 추세 강도**: Hurst 지수 + ADX 이중 확인으로 레짐 판단 정확도 향상
- **멀티타임프레임 모멘텀**: 5d/10d/20d/60d 가중 혼합 (n_days에 따라 가중치 변동)
- **피보나치 자석 효과**: 0.236/0.382/0.500/0.618/0.786 레벨 추가
- **거래량 프로파일 drift**: 최근 거래량 패턴 + OBV 방향성 반영
- **캔들 패턴 반영**: 망치형/장악형 등이 초기 드리프트에 가산
- **레버리지 효과**: 하락 시 변동성 비대칭 증가
- **적응형 GARCH**: 레짐별 alpha/beta 조정 (추세장: 관성 강함, 횡보장: 충격 반응 큼)
- **VWAP 기준점**: 20일 거래량가중평균가격을 지지/저항 레벨로 추가

### 상세 로직
1. **ADX 계산**: 14일 ADX → 추세 강도 + +DI/-DI로 방향 판별
2. **레짐 분류 (이중 확인)**: ADX≥30 & Hurst>0.50 = 강한추세, ADX<20 & Hurst<0.50 = 횡보
3. **변동성 분석**: 과거 120일 → 연환산 변동성·레버리지 비율(하락변동성/상승변동성) 계산
4. **드리프트 구성**: score_drift + 멀티TF 모멘텀 + 거래량 drift + 캔들 패턴 drift + ADX 부스트
5. **몬테카를로 루프**: 50개 경로 × n_days, 각 경로에 GARCH + S/R 자석 + 평균회귀 적용
6. **앙상블 집계**: 고가=60th 퍼센타일, 저가=40th 퍼센타일, 시/종가=중앙값

## 지지/저항선 탐지 (v2)

- **피봇 포인트 기반**: 다중 타임프레임(5/10/20봉) 피봇 고·저점 수집
- **종합 점수로 가장 유의미한 1개씩만 선택**:
  - 터치 횟수 (40%): 같은 가격대를 여러 번 터치할수록 강한 레벨
  - 거래량 집중도 (30%): 해당 가격대에서 거래량이 많을수록 강함
  - 최근성 (20%): 최근 형성 레벨 우선
  - 현재가 근접도 (10%): 가까울수록 즉시 영향력
- **유효 범위**: 현재가 위/아래 1%~20% 이내

## 피보나치 되돌림 (`_compute_fibonacci_levels`)

- **기준 기간**: 최근 120일 고점/저점 (swing_high, swing_low)
- **방향 결정**: 현재가가 범위 상단 40% 이상이면 하락 되돌림, 하단 60% 이하이면 상승 복귀
- **레벨**: 0.236 / 0.382 / 0.500 / 0.618 / 0.786
- **차트 표시**: 골드 계열 점선(`LineStyle.Dashed`), 레이블 `Fib XX%`
- **API 응답**: `/api/stock/{ticker}` 초기 조회 시 `fibonacci` 필드로 포함
  - `swing_high`, `swing_low`, `levels` (비율→가격 dict), `direction` ("up"|"down")

## 수렴→돌파→눌림목 패턴 감지

스코어링(`_unified_score`)과 예측(`_generate_predicted_candles`) 양쪽에서 처리:

- **돌파 감지**: 최근 10일 내 BB 상단/박스 상단을 종가 기준으로 돌파 + 거래량 20일 평균의 1.8배 이상
- **눌림목 판정**: 돌파 후 되돌림이 돌파 레벨의 -3%~+5% 이내 + 되돌림 중 거래량 감소
- **스코어링 반영**: 눌림목 상태 = structure_pts +7 (최적 매수 타이밍), 돌파 직후 = +5
- **예측 반영**: 돌파 후 눌림목이면 저항선 반발력을 1/3로 완화 (건전한 조정을 되돌림으로 오인하지 않도록)
- **시너지**: breakout_pullback이 다른 반등 신호와 겹치면 시너지 보너스 가산

## 커스텀 지표 설명

- **수박지표**: 볼린저밴드 스퀴즈(밴드폭 수축) + 이동평균선 수렴(5/20/60/224일 편차 < 2%)이 동시에 발생하면 추세 전환 임박 신호
- **계단지표**: ATR의 0.5배를 그리드 크기로 하여 종가를 계단화. 동일 레벨 3회 이상 유지 + 레벨 상향 시 상승 확인

## 추천 종목 필터링 기준

`_compute_recommendations_sync()`:
1. 시가총액 ≥ 500억 원 (소형주 기술적 신뢰도 낮음)
2. 거래대금 ≥ 20억 원 (유동성 부족 종목 실매매 불가)
3. 위 두 조건 데이터 없으면 리스팅 순서로 fallback

## 설계상 결정 (의도적)

- **예측 seed 결정론적**: `(last_close * 100 + date_hash)` 기반 → 같은 종목/날짜면 항상 동일한 50개 경로 생성. 재현성을 위한 의도적 설계.
- **SMA20 평균회귀**: `mean_revert += -dist_sma20 * 0.10` — 음수 부호가 SMA20 방향으로 당기는 것을 의미 (올바른 방향).
- **스크롤 추가 로드 시 fibonacci 미계산**: 스크롤 로드는 OHLCV only, 초기 로드 시에만 fibonacci 계산.

## 스토캐스틱 오실레이터 (14-3-3 파라미터)

`_unified_score()` 내 모멘텀 카테고리에 추가됨:
- `%K` = (종가 - 14일 최저) / (14일 최고 - 14일 최저) × 100, 3일 평활화
- `%D` = %K의 3일 이동평균 (시그널선)

| 조건 | 점수 | 설명 |
|------|------|------|
| %K ≤ 20 + %D ≤ 20 + RSI ≤ 35 | +5 | 이중 과매도 (강한 반등 신호) |
| %K ≤ 20 + %D ≤ 20 | +3 | 스토캐스틱 과매도 |
| %K ≥ 80 + %D ≥ 80 + RSI ≥ 65 | -5 | 이중 과매수 (강한 조정 경고) |
| %K ≥ 80 + %D ≥ 80 | -3 | 스토캐스틱 과매수 |
| %K가 %D를 상향 돌파 + %K < 50 | +3 | 바닥에서 반등 전환 |
| %K가 %D를 하향 돌파 + %K > 50 | -2 | 고점에서 하락 전환 |

스토캐스틱 과매도 상태는 `reversal_flags`에 포함되어 시너지 보너스에도 기여.

## 일목균형표 (`_compute_ichimoku`)

- **전환선**: 9일 중간값 `(9일 고가 max + 9일 저가 min) / 2`
- **기준선**: 26일 중간값
- **선행스팬1**: `(전환선 + 기준선) / 2` (현재 시점 기준 반환)
- **선행스팬2**: 52일 중간값
- **구름(Kumo)**: 선행스팬1 vs 선행스팬2 범위 — 양운(녹색) vs 음운(빨강)
- **후행스팬**: 현재 종가 (차트에서 26봉 이전에 표시하는 방식은 미구현 — 현재값만 전달)

### 스코어링 반영 (추세 카테고리)
| 조건 | 점수 |
|------|------|
| 구름 위 + 전환선>기준선 + 양운 | +4 |
| 구름 위 | +2 |
| 구름 아래 + 전환선<기준선 + 음운 | -4 |
| 구름 아래 | -2 |
| 구름 안 (중립) | 0 |

### 차트 표시 (CandleChart.tsx)
- 전환선: 빨간 실선 `#f87171`, 레이블 `전환`
- 기준선: 파란 실선 `#60a5fa`, 레이블 `기준`
- 구름 상단/하단: 점선, 양운=녹색 `rgba(34,197,94,0.5)` / 음운=빨간 `rgba(239,68,68,0.5)`
- API 응답 필드: `ichimoku` (`/api/stock/{ticker}` 초기 조회 시만 포함)

## 다이버전스 감지 (개선된 find_peaks 버전)

기존 단순 전반부/후반부 비교 → **scipy.signal.find_peaks 기반 실제 피크 쌍 탐색**으로 교체:

```python
# RSI 상승 다이버전스: 최근 60일에서 저점 쌍 추출 (distance=8봉, prominence=1σ)
trough_idx, _ = find_peaks(-c_arr, distance=8, prominence=c_arr.std() * 0.3)
# 마지막 두 저점 i1, i2: 가격↓ & RSI↑ → 불리쉬 다이버전스
```

| 신호 | 점수 | 조건 |
|------|------|------|
| RSI 상승 다이버전스 | +5 | 가격 저점↓ & RSI 저점↑ (find_peaks) |
| RSI 하락 다이버전스 | -4 | 가격 고점↑ & RSI 고점↓ |
| MACD 상승 다이버전스 | +4 | 가격 저점↓ & MACD 히스토그램 저점↑ |
| MACD 하락 다이버전스 | -4 | 가격 고점↑ & MACD 히스토그램 고점↓ |

## 남아있는 개선 포인트

### 중기 개선 (P1)
1. **OBV 차트 표시**: 스코어링엔 쓰이지만 차트에 미표시 (하단 패널 추가 필요)
